# Checks

Use explicit checks to classify execution results after values have been extracted or transformed.

All entries use the selected block's settings panel. The focused configuration guidance below describes the fields to review before using the block.

## Key Check

- **Rust type:** `KeyCheck`
- **Availability:** Palette
- **Purpose:** Evaluates configured conditions and sets a named result status such as Success, Fail, Ban, Retry, Error, or ToCheck.
- **Configure:** Define non-empty condition groups, select the resulting status, and use valid numeric operands for numeric comparisons.
